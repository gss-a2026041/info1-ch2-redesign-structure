## 📖 教科書で押さえるべき重要ポイント（学習メモ）

実習を始める前に，教科書（実教『最新情報Ⅰ』94〜99ページ）に掲載されている情報デザインの基本概念を確認しましょう．

### 1．情報の分類・整理（LATCHの原則）
アメリカのデザイナーであるリチャード・ソール・ワーマンが提唱した，すべての情報を分類・整理するための代表的な5つの観点（それぞれの頭文字をとって **LATCH** と呼びます）です．
- **L**ocation（位置）：建物のフロアマップや日本地図，路線図など，物理的な「場所」で整理する方法．
- **A**lphabet（アルファベット）：国語辞典の五十音順やスマートフォンのアドレス帳など，規則的な「名前順」で整理する方法．
- **T**ime（時間）：歴史の年表やスケジュール帳，列車の時刻表など，情報の「時間軸や前後関係」に沿って整理する方法．
- **C**ategory（分野）：図書館の本棚やECサイトの商品ジャンルなど，情報の「類似性や共通点（カテゴリ）」で整理する方法．
- **H**ierarchy（階層）：重要度によるランキングや，Webページにおける「見出し」のレベルなど，特定の基準による「優先度やレベルの差」で整理する方法．

★ 今回扱う「HTMLの見出しタグ（`h1`〜`h3`）」は，このLATCHの中の **Hierarchy（階層）** を用いて情報の構造と優先順位を表現する，最も基本的かつ強力な情報デザイン手法です．

### 2．レポートの基本構成とHTML構造化
WebページのHTMLタグの階層構造は，学校や大学で書く **「レポート・論文の構成」** とまったく同じ考え方です．
- **レポートの構成：** タイトル ➔ 章 ➔ 節 ➔ 項 ➔ 本文
- **HTMLの構成：** `<h1>` ➔ `<h2>` ➔ `<h3>` ➔ `<h4>` ➔ `<p>`
見出しタグを正しい順番（`h1` ➔ `h2` ➔ `h3`）で使うことで，論理的に整理された読みやすい構成になります．

### 3．情報バリアフリーとアクセシビリティ
なぜ，文字の大きさや色を変更（ビジュアルの調整）するだけでなく，正しい「タグ」を選ぶ必要があるのでしょうか？
- **音声読み上げソフトへの対応：** 視覚障がいのある人が使う音声読み上げソフトは，HTMLの「タグ」を頼りに文書の構造を理解し，読み上げる順序や重要度を判断します．
- **検索エンジン（SEO）への対応：** 検索ロボットもタグを解析してページの中身を理解します．
見た目の変更（CSS）を行う前に，まず正しいタグ（HTML）で文書の論理構造を定義することが，情報バリアフリー（アクセシビリティ）の第一歩です．

## 🛠 構造化の要件（ミッションクリア基準）

以下の要件を満たすように `index.html` の `<body>` タグ内を書き換えてください．

1. **見出しの階層化：**
   - 最も重要なタイトルには `<h1>` を使います．
   - 各セクション（「イベント詳細」「持ち物リスト」「タイムスケジュール」）の見出しには，`<h2>` を使います．
2. **箇条書きリストの作成：**
   - 「持ち物リスト」の各項目を，順序なしリストタグ（`<ul>` と `<li>`）を使って正しく構造化します．（`<br>` による改行は使用しません）
3. **表の作成：**
   - 「タイムスケジュール」の時間・内容・担当者のデータを，表タグ（`<table>`，`<tr>`，`<th>`，`<td>`）を使って整理します．（`<br>` による改行や空白スペースによる調整は使用しません）
   - 表の1行目（ヘッダー）には `<th>` を，データ行には `<td>` を使用してください．

- 「プレビュー画面で表が崩れて表示されるのですが，どのタグの記述が間違っているか教えてください．」

## 💾 提出と自動評価の確認手順

### Step 1. AIとのチャット履歴の保存
AIと対話した場合は，チャットパネルの右上「…」から **「チャットのエクスポート」** を選択し，`copilot-chat-history.json` という名前でこのフォルダ直下に保存します．（対話していない場合も，ファイルが空またはファイル自体がなくても自動評価は実行されますが，評価対象となります）

### Step 2. Gitによるコミットとプッシュ
左側のソース管理パネル（ブランチアイコン）を開き，以下の手順を実行します．
1. 変更したファイル（`index.html`，`copilot-chat-history.json`）の横にある **「＋」（ステージ）** をクリックします．
2. メッセージ入力欄に **`feat: structure index.html`** と入力します．
3. **「コミット」** ボタンをクリックします．
4. **「変更の同期（プッシュ）」** をクリックして，GitHub（クラウド）に送信します．

### Step 3. 自動評価（Issue）の確認
1. ブラウザで自分のGitHubリポジトリページを開きます．
2. 画面上部にある **「Issues」** タブをクリックします．
3. **「【自動評価】第5回課題 ルーブリックフィードバック」** というIssueが届いているか確認します（プッシュから約1分で届きます）．
4. 中身を開き，すべて🟢（合格）になっていればミッション完了です！
5. 🔴（要修正）がある場合は，アドバイスを読んでCodespacesでコードを修正し，再度 **Step 2（コミット＆プッシュ）** を実行してください．Issueが自動で再評価され，内容が上書き更新されます．
